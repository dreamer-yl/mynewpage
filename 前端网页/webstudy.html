AJAX技术

使用ajax技术很多都要跨域，因此先介绍常见跨域知识

（一）JSONP

JSONP个人理解就是利用src不限制跨域的特点，在函数中包含数据，这需要服务端配合，操作性不高

（二）iframe

iframe通过内嵌页面打开跨域窗口，个人获取数据不是特别方便,如获取淘宝时间(查看浏览器控制台就会发现，会出现黄色提醒，因为这本是json数据，用网页形式显示不佳)

（三）CORS

CORS这是最普遍也是最方便的，只需要服务端设置allow origin为全域就可以跨域啦，浏览器先发送一个option试探，如果服务端同意就继续正常请求，主流浏览器当涉及跨域问题时都支持自动试探cors


点击上面的按钮，毫无反应，去浏览器控制台看可发现浏览器禁止了跨域行为，当然有很多支持的，比如图灵机器人

（四）XMLhttprequest

XMLhttprequest是比较原生的网页异步请求的方式，首先需要创建一个对象，再open（）函数设置参数，最后send（）函数内置函数发送，大多网页介绍的十分晦涩南东方，鄙人写了一个http函数方便理解。


            function httpRequest(url, method, async，headers, data, content_type, callback) {
                var xhr = new XMLHttpRequest();
                async = !!async
                if (!data) {
                    data = null
                }
                else{
                    if(data&&content_type=='application/json'){
                        data=JSON.stringify(data)//xmlhttprequest只接受字符串格式
                    }
                }
                xhr.open(method, url, async);
                if (content_type) {
                    xhr.setRequestHeader('Content-Type', content_type)
                } else if (method == "POST" && !content_type) {
                    xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded;charset-UTF-8')//post一般不是json就是表单
                }
                if (headers && typeof headers === 'object') {
                    for (key in headers) {
                        xhr.setRequestHeader(key, headers[key])
                    }
                }
                else{
                    alert("请以对象形式传入头部信息")
                    return
                }
                if (callback && async) {//如果异步，需要监听加载完成才执行，不然会unexpect报错
                    xhr.addEventListener("load", function () {
                        callback(xhr)/回调函数传入xhrobj
                    })
                }
                xhr.send(data)
                if (!async) {
                    if (callback) {
                        callback(xhr)//回调函数传入xhrobj
                    }
                    return xhr
                }
            }
